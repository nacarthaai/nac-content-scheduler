# NAC MEMORY FILE v1.0

## Core Rule

For EVERY video generated by the NAC system:

1. Never use narration-only content.
2. Every spoken line must have matching visual support.
3. Every stock, company, strategy, event, chart, indicator, news item, or market concept mentioned must have relevant visuals generated or retrieved.
4. Viewers should never hear a concept without seeing it.

---

## Stock Content Rule

When narrating:

* What a stock did today
* Why a stock moved
* Earnings results
* Technical analysis
* Trading strategies
* Options strategies
* Market news

The system must automatically generate:

### Visual Layer A

* Stock chart
* Candlestick chart
* Volume chart
* Relevant indicators

### Visual Layer B

* Company visuals
* Headquarters
* Products
* CEO photos (when appropriate)
* Industry visuals

### Visual Layer C

* Strategy explanations

Examples:

If narration says:

"NVDA broke above resistance today."

Show:

* Resistance line
* Breakout animation
* Chart example

If narration says:

"Micron benefited from AI demand."

Show:

* AI data centers
* Servers
* Memory chips
* Semiconductor manufacturing visuals

If narration says:

"This is a bull flag pattern."

Show:

* Bull flag illustration
* Real chart example
* Animated explanation

## Educational Content Rule

When teaching:

* Trading
* Investing
* Options
* Technical analysis
* AI
* Business

Every concept must include:

* Diagram
* Visual explanation
* Infographic
* Supporting image

No talking-head-only segments.

---

## Character Consistency Rule

Use the master NAC character sheet.

Maintain:

* Same face
* Same identity
* Same age
* Same hairstyle
* Same proportions

Allowed roles:

* Trader
* Teacher
* News anchor
* Analyst
* Business presenter

---

## Engagement Rule

Every 3–5 seconds:

Show one of:

* New image
* Chart change
* Zoom
* Animation
* Highlight
* Infographic
* Relevant visual

No static screens for extended periods.

---

## Automation Rule

For every script:

Step 1: Generate narration.
Step 2: Break narration into scenes.
Step 3: Generate matching visuals.
Step 4: Verify every sentence has supporting visuals.
Step 5: Generate final video.
Step 6: Generate thumbnail.
Step 7: Publish.

---

## Quality Rule

Before publishing:

Ask: "Does every statement have a visual?"

If not: Generate additional images before rendering.

Never publish narration-only videos.
